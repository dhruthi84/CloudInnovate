Our approach for cloud adoption

Today’s CIOs need to leverage the power, speed, agility and cost advantages of public clouds. At the same time, they must preside over rapid transformation of their existing IT infrastructure and application landscapes. In order to rapidly move applications to public clouds, and modernize current portfolios to exploit the capabilities of hybrid, multi-cloud landscapes, our clients need outcomes which are proven, reliable and can be delivered repeatedly. Through applying what we have learned through hundreds of client engagements, IBM has developed an end-to-end methodology: Cloud Innovate. The Cloud Innovate methodology incorporates tools, technologies, accelerators and cognitive capabilities to deliver effective, efficient transformations in a way no competitor can match.

Multiple demand profiles are addressed via one method, one tool set, one process
Powered by cognitive, the Cloud Innovate methodology provides a roadmap of the overall operating environment, identifying which applications can immediately be migrated, which can be migrated with additional development, and which applications are not suitable to shift to the cloud.

Cross IBM solution design and engagement model
We are delivering the best of IBM by launching a cross-brand Solution Engineering Center of Excellence. This approach will ensure a virtual team of cloud unit and GBS experts who systematically reach out to the GTS solutioning teams and processes.

Modernization to cloud

How is Cloud Innovate Migrate to cloud different than server replication, scripting, templates and manual migration?

Cloud Innovate -> Migrate to cloud is a predictable migration solution that automates the application information collection, identifies the application dependency and migrate of application workload and their associated metadata/configuration into cloud. Step-by-step migration methods offers automated migration of workloads without any pre-processing on the source environment or post-processing on the target cloud to minimize business disruption.

What is the basic difference between migrating application workloads and servers? Aren't they the same?

Application installed in servers containing the OS, the applications and platforms (e.g. application servers, databases) associated with the workload. Application Workloads typically include the entire suite (servers), data storage (data may be stored on attached and/or networked storage) and the network and system configurations associated with the workload. Our Migrate to cloud method ensures application entire workload can be migrated seamlessly with minimum business intervention.

Does Cloud Innovate Migrate to cloud supports migrate to SaaS?

Yes – Our analytical tools supports to identify the application source pattern to create the right target cloud pattern and execute the migrate process according to the best suitable pattern to avoid business disruption while taking vantage of cloud.

Does Cloud Innovate Migrate to cloud require a proprietary environment to import application information into prior to migration to cloud platform ?

It does need a transient environment to support the conversion before application workloads are deployed into the target cloud platform. This is only required if application workload needs conversion e.g. platform upgrade or remediation etc. Our migrate toolset is capable of hosted either in client environment or target cloud platform to pull the application binaries, data accordingly to network connection used between source and target cloud connection

What does it mean to migrate a server live?

It means that production servers can be migrated without having to take them down out of production during the migration. Once the migration is complete and the workload is configured and tested on the target cloud environment, the client can set up a change window to cut-over to the new VM deployed in the target data cloud.

What Platforms Does Migrate to cloud support?

It supports migration from physical, virtual and cloud platforms to cloud platforms. Supported OS, hypervisors and Clouds are:
and Source OS – Windows 2003/2008/2012, Linux, AIX
and Target Clouds – IBM Bluemix, AWS VPC, VMWare vCloud, Azure or private cloud platform.

Do I need to install anything discover application information ? What about agents?

Our discovery toolset is an agentless technology, so you will not need to install agents. Yes, our migration capabilities are more geared with enterprise application assessment, creating the right migration roadmap to minimise client risk. Our large toolset supporting migration assessment, planning and execution is end-2-end automated to get a most predictable outcome in a short period of time.
